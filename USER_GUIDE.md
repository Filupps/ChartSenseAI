#  Руководство пользователя ChartSenseAI

ChartSenseAI — это веб-сервис для анализа диаграмм и блок-схем с автоматическим извлечением алгоритмов и генерацией кода.

---

##  Быстрый старт

1. Откройте веб-интерфейс приложения в браузере  
2. Выберите нужный режим работы:
   - **Диаграмма → Алгоритм**
   - **Алгоритм → Диаграмма**

---

##  Диаграмма → Алгоритм

Этот режим используется для анализа изображений блок-схем.

### Как пользоваться:
1. Перейдите на вкладку **«Диаграмма → Алгоритм»**
2. Загрузите изображение диаграммы (PNG / JPG)
3. Нажмите кнопку **«Анализировать»**
4. Дождитесь завершения обработки

### Что вы получите:
- Распознанные элементы диаграммы
- Извлеченный текст из блоков
- Структурированный алгоритм
- Возможность:
  - скачать результат в JSON
  - сгенерировать код (Python / псевдокод)

### Редактирование алгоритма
   - Нажмите кнопку **"Редактировать"** (иконка карандаша) в карточке алгоритма
   - Для **изменения шага**: кликните по тексту шага, внесите изменения, нажмите ✓
   - Для **добавления шага**: нажмите кнопку "+" рядом с шагом — новый шаг добавится после него
   - Для **удаления шага**: нажмите кнопку "×" рядом с шагом
   - Для **завершения редактирования**: нажмите кнопку "Готово"

---

## Алгоритм → Диаграмма

Этот режим позволяет создать диаграмму по текстовому описанию алгоритма.

### Как пользоваться:
1. Перейдите на вкладку **«Алгоритм → Диаграмма»**
2. Введите описание алгоритма или вставьте текст
3. Нажмите кнопку **«Сгенерировать диаграмму»**
4. Получите визуальную диаграмму (SVG / PNG)

Диаграмма автоматически создаётся с использованием PlantUML.

##  История и сессии

- Все результаты сохраняются в базе данных
- История привязана к пользовательской сессии
- Можно просматривать:
  - предыдущие обработки
  - сгенерированные алгоритмы
  - созданные диаграммы

---

##  Поддерживаемые форматы

**Вход:**
- PNG, JPG, JPEG — изображения диаграмм
- Текстовое описание алгоритма

**Выход:**
- JSON — структурированный алгоритм
- Python / псевдокод
- SVG / PNG — диаграммы

---

## Возможные ограничения

- Качество распознавания зависит от чёткости изображения
- Сложные или нестандартные диаграммы могут требовать ручной корректировки
- Генерация кода выполняется автоматически и может требовать доработки

---
##  Решение проблем

**Ошибка подключения к БД:**
- Проверьте, что MySQL запущен
- Убедитесь, что база данных создана
- Проверьте параметры в `.env`

**Модель не загружается:**
- Убедитесь, что файл модели существует в `models/`
- Проверьте размер файла (не должен быть пустым)
- Система автоматически скачает fallback модель

**OCR не распознаёт текст:**
- Установите Tesseract с русским языком
- Проверьте `TESSERACT_LANG` в настройках

## Советы

- Используйте контрастные и четкие изображения диаграмм
- Старайтесь избегать пересекающихся стрелок и неклассических направлений
